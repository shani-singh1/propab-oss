name: anti-rediscovery-and-novelty-check
description: Check a claim against what is already established — reproducing a known value is a rediscovery, not a discovery
phase: evidence
scope: core
priority: 35
---
Reproducing a value that is already known, tabulated, or proven is a REDISCOVERY. It can be
worth doing as a sanity check on your pipeline, but it is NOT a finding and must never be
reported as one. Before you propose a claim, and again before you confirm a result, test it
against the established record.

1. **Locate the claim on the map of what is known.** Sort the target into one of:
   - **Established / tabulated / proven** — the answer already exists in the literature or a
     reference table. Matching it is a rediscovery; the correct verdict is "reproduces the
     known value" with an explicit citation, never "discovered".
   - **Contested / conjectured** — a value or claim proposed but not settled. Adding
     confirming or refuting evidence here is genuine contribution.
   - **Open / unknown** — no accepted answer. This is the only place a first correct result
     is a discovery.
   If you cannot say which bucket a claim falls in, you have not yet checked novelty — do that
   before spending effort.

2. **Compare to the best-known baseline, honestly.** State the current best-known value or
   result for the target and report your result RELATIVE to it: below / matches / exceeds.
   "Matches best-known" is a validation of your method, not a new result. Only "exceeds the
   best-known" (or resolves something previously open) is a candidate finding — and only if it
   survives adversarial testing.

3. **A rediscovery reached by asserting the known answer is worthless — and dangerous.** If a
   result "confirms" a tabulated value because the value was fed in, looked up, or assumed,
   that is circular. A rediscovery only has sanity-check value when the object was genuinely
   recomputed from scratch and the recomputed answer happened to agree; the answer must fall
   out of the computation, never into it.

4. **Do not dress a settled question as open.** Restating a known result with different
   wording, a renamed quantity, or a swapped parameter of a settled finding is not novelty.
   Ask: if a domain expert saw this, would they say "we already knew that"? If yes, it is a
   rediscovery regardless of how it is phrased.

5. **Novelty is necessary but not sufficient.** A claim that is genuinely new but unverified is
   still just a conjecture. Novelty earns the right to test; only adversarial confirmation earns
   the word "finding". Never trade rigor for the appearance of originality.

The bar: a result is reportable as a discovery only if (a) it is not already established, AND
(b) it beats or resolves the best-known baseline, AND (c) it survives an adversarial test. Fail
any one and it is a rediscovery, a restatement, or an unverified guess — label it as such.
